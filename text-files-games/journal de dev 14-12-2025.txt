🗓 2025-12-14 — Génération jeux Kikongo (mots mêlés + mots croisés)

1. Rappel structure Java / DB

Utilisation du module backend-java avec Maven (mvn compile, mvn exec:java).

Connexion DB centralisée via DbConfig (profil local utilisant config/db-local.properties).

Base Kikongo : 6i695q_lexikongo (tables words, verbs, word_meanings, verb_meanings, slugs, etc.).

1.1. Mots croisés Kikongo — NOMS

Outil CLI : com.longoka.games.app.KikongoNounCrosswordJsonExportTool

Service sous-jacent :

KikongoCrosswordService (mode "nouns").

Génère des grilles à partir d’un “sac” de WordToFind issus des mots Kikongo.

Commande utilisée :

mvn exec:java -Dexec.mainClass=com.longoka.games.app.KikongoNounCrosswordJsonExportTool


Paramètres principaux :

puzzleCount = 48

rows = 12, cols = 12

maxEntries = 12

meaningLanguageCode = "fr"

Sortie JSON :

target/kikongo-nouns-crossword-book.v2.json (puis v3/v4 selon itérations).

Format longoka.crossword.pack.v1 :

pack.language = "kg"

pack.packId = "kg-cross-nouns-<date>"

Chaque puzzle : entries[] avec translation, translationEn, phonetic, slug, extraInfo (classe nominale), semanticTags, meta.nominalClasses, meta.semanticDomains.

1.3. Mots mêlés Kikongo — NOMS

Service : com.longoka.games.lexikongo.KikongoWordSearchService

Récupère des LexWord depuis la DB.

Outil JSON : com.longoka.games.app.KikongoNounWordSearchJsonExportTool

Transforme un WordSearchPuzzle en WordSearchJsonModels.PackV1.

Déduplication des entrées par base (singulier/pluriel/racine).

Commande :

mvn exec:java -Dexec.mainClass=com.longoka.games.app.KikongoNounWordSearchJsonExportTool


Sorties successives :

target/kikongo-nouns-wordsearch-book.v3.json

Puis v4 avec amélioration des données.

Champs gérés par EntryV1 :

base, display, translation (FR fusionné), slug, partOfSpeech, extraInfo (classe nominale), phonetic, translationEn, semanticTags.

meta inclut : source = "lexikongo", meaningLanguage = "fr", nominalClasses.

1.5. Mots mêlés Kikongo — MIXTE (noms + verbes)

Service dedicated : KikongoMixedWordSearchService

Ouvre une connexion Kikongo via DbConfig.openKikongoLexConnection().

Utilise :

LexikongoWordRepository (noms)

LexikongoVerbRepository (verbes)

Paramétrage :

nounTarget = maxWords / 2, verbTarget = maxWords - nounTarget.

Sur-sélection (* 3) pour laisser de la marge au générateur.

Remplissage via :

fillFromNouns(...) → WordToFind avec :

baseForm = singular/plural/root

translationFr = getFrenchMeaningsJoined() (fallback sur ancienne logique)

translationEn = getEnglishMeaningsJoined()

extraInfo = nominalClass.className

phonetic, slug

fillFromVerbs(...) → WordToFind avec :

baseForm = v.getName()

translationFr, translationEn, phonetic, slug.

Générateur :

WordSearchGenerator.generatePuzzle("kg", "Mots meles Kikongo (noms + verbes)", "Noms + verbes", rows, cols, wordsToFind).

Outil JSON : KikongoMixedWordSearchJsonExportTool

Gère déduplication par base, fusion des traductions, devinette de semanticTags via SemanticTagger.guessTags.

Sortie : target/kikongo-mixed-wordsearch-book.v2.json.

1.6. Mots croisés Kikongo — MIXTE (noms + verbes)

Nouveau service : KikongoMixedCrosswordService

S’inspire très fortement de KikongoCrosswordService.buildCrosswordFromWords.

Étapes :

Construction d’une liste MixedEntryCandidate à partir de :

LexikongoWordRepository (noms)

LexikongoVerbRepository (verbes)

Sélection mixte (sans dupliquer les baseForm / answer).

Génération de la grille via la même logique de placement :

Grille 12×12 remplie avec #.

Placement du premier mot en ACROSS.

Puis placement par croisements (tryPlaceWithCrossing), fallback horizontal/vertical (tryPlaceSimple).

Construction de CrosswordJsonModels.PuzzleV1 :

id = "kg-cross-mixed-" + index

mode = "mixed", title = "Mots croisés Kikongo (mixed)", theme = "noms + verbes".

entries[] avec :

id, number, direction, row, col, answer

display, clue = translation FR

translation, translationEn, phonetic, slug, partOfSpeech, extraInfo, semanticTags.

meta.semanticDomains = union des tags, meta.nominalClasses = union des extraInfo des noms.

Outil : KikongoMixedCrosswordJsonExportTool

Commande :

mvn exec:java -Dexec.mainClass=com.longoka.games.app.KikongoMixedCrosswordJsonExportTool


Sortie :

target/kikongo-mixed-crossword-book.v2.json

48 grilles 12×12, mixant noms + verbes.

🗓 2025-12-15 — Passage au Lingala (mots mêlés + croisés)
2.1. Config DB Lingala

DbConfig.openLingalaLexConnection() ajouté.

Host / port / user / pass paramétrables via :

lex.ln.host, lex.ln.port, lex.ln.user, lex.ln.pass, lex.ln.name

ou variables d’environnement LEX_LN_DB_*.

DB par défaut : 6i695q_lexilingala.

Hypothèse importante :

Même schéma que Lexikongo : mêmes tables (words, verbs, *_meanings, slugs, etc.).

2.2. Repositories Lingala

Création des repositories analogues à ceux du Kikongo, mais branchés sur openLingalaLexConnection() :

LingalaWordRepository (basé sur LexikongoWordRepository).

LingalaVerbRepository (basé sur LexikongoVerbRepository).

Logique identique :

Sélections de mots approuvés, filtrés par langue de meaning ("fr" et "en"), fusion FR/EN, phonétique, slug, classes nominales, etc.
